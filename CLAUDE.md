# Персональный сайт-портфолио

## Обзор проекта
Создать современный минималистичный сайт-портфолио для Frontend team lead. Сайт должен демонстрировать профессиональный опыт, технические лидерские качества и проекты в чистом tech-стиле.

## Технологический стек
- **Фреймворк:** Next.js 14+ (App Router)
- **Язык:** TypeScript
- **Стилизация:** Tailwind CSS + shadcn/ui компоненты
- **Деплой:** Vercel (бесплатный тариф)
- **Контент:** Markdown файлы для проектов

## Структура сайта
Одностраничное приложение с плавной прокруткой между секциями:
1. **Hero** - Краткое представление и призыв к действию
2. **Обо мне** - Детальное описание бэкграунда и экспертизы
3. **Опыт работы** - Профессиональные достижения на текущей работе
4. **Пет-проекты** - Витрина личных проектов
5. **Контакты** - Социальные ссылки и контактная информация

## Требования к дизайну

### Визуальный стиль
- **Тема:** Только тёмная тема
- **Эстетика:** Минимализм в tech-стиле
- **Цвета:** Тёмный фон с акцентными цветами для CTA и подсветки
- **Типографика:** Чистые современные sans-serif шрифты
- **Макет:** Просторный, много воздуха между секциями

### Анимации
- Плавная прокрутка между секциями
- Fade-in анимации для элементов при появлении в viewport
- Тонкие hover-эффекты на интерактивных элементах
- Transition-анимации для изменения состояний

### Адаптивный дизайн
- Mobile-first подход
- Breakpoints: mobile (< 768px), tablet (768-1024px), desktop (> 1024px)
- Touch-friendly взаимодействия для мобильных

### Секция "Опыт работы"
Формат: карточки с ключевыми достижениями на текущей позиции

**Текущая роль:** Frontend Team Lead

**Управление командой:**
- Руковожу командой из 3 frontend разработчиков
- Внедрил структурированные процессы команды: регулярные 1-on-1, планирование через диаграммы Ганта (Excel)
- Синхронизация с дизайнерами и продуктовой командой через общую табличку - получили прозрачный бэклог

**Ключевые технические достижения:**

1. **E2E тестирование с нуля**
   - С нуля предложил и разработал систему E2E тестов на базе Playwright
   - Придумал объектную модель для тестов
   - Настроил CI для автоматического запуска (изначально Jenkins, сейчас мигрируем на GitLab)
   - Результат: более 2000 тестов, до этого CI тестирования не было вообще

2. **Интеграция системы чатов**
   - Реализовал через WebSocket
   - Функционал: личные чаты, групповые чаты, отправка файлов
   - Теперь можно использовать компонент чата из прикладного кода

3. **PDF просмотрщик для мобильных и десктопа**
   - Разработал просмотрщик PDF, работающий корректно и на мобильных устройствах
   - Решал сложности с плавным масштабированием и подгрузкой контента
   - Оптимизирован для различных размеров экранов

4. **Инфраструктурные улучшения**
   - Сэкономил ~100 человеко-часов в месяц за счёт стабилизации инфраструктуры
   - Предотвращение блокировок релизов

**Структура данных проектов (Markdown файлы в `/content/projects/`):**

Каждый файл проекта должен содержать frontmatter:
```markdown

description: "Краткое описание"
tags: ["тег1", "тег2", "тег3"]
link: "https://github.com/..." (опционально)
status: "in-progress" | "completed" | "closed"
featured: true | false
type: "work" | "pet"
---

Детальное описание проекта в markdown...
```

     - Профили с верификацией принадлежности к конкретному вузу
     - Супер-лайки
     - Система верификации пользователей
   - Статус: Закрыт
   - Теги: Node.js, TypeScript, Telegram Bot, Startup, Product

## Техническая реализация

### Навигация
- Фиксированный хедер с навигацией по якорям
- Ссылки на каждую секцию (Обо мне, Опыт, Проекты, Контакты)
- Подсветка активной секции
- Мобильное hamburger-меню

### Система проектов
- Проекты хранятся в `/content/projects/*.md`
- Используем `gray-matter` для парсинга frontmatter
- Отображение проектов в grid layout (карточки)
- Две отдельные секции: "Опыт работы" (type: "work") и "Пет-проекты" (type: "pet")
- Клик для раскрытия/модальное окно с полными деталями
- Легко добавлять новые проекты, создавая новые .md файлы

### Производительность
- Оптимизация изображений (Next.js Image компонент)
- Code splitting
- Lazy loading для контента ниже первого экрана
- Быстрая начальная загрузка

### SEO
- Meta теги для социального шеринга
- Семантический HTML
- Правильная иерархия заголовков
- Alt текст для изображений
